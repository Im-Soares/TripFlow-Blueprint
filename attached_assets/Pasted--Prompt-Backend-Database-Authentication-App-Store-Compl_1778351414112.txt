# Prompt — Backend, Database, Authentication & App Store Compliance for TripFlow

Quero transformar o TripFlow numa aplicação totalmente funcional, pronta para produção e preparada para publicação na Apple App Store.

A aplicação já possui estrutura visual/frontend.

O objetivo agora NÃO é recriar o projeto.

O objetivo é:

* ligar todo o frontend a um backend real
* criar toda a base de dados necessária
* implementar autenticação funcional
* implementar persistência real de dados
* implementar colaboração e partilha de viagens
* garantir conformidade com os requisitos da Apple App Store
* melhorar estabilidade, segurança e arquitetura

A aplicação deve ficar production-ready.

---

# 1. STACK TÉCNICA (ESCOLHER A MELHOR OPÇÃO)

Usar uma stack moderna, escalável, rápida e altamente compatível com aplicações SaaS modernas.

## Recomendação principal

Frontend:

* Next.js
* TypeScript
* Tailwind CSS
* Framer Motion
* Zustand ou Redux Toolkit
* React Query / TanStack Query

Backend:

* Next.js App Router + Server Actions + API Routes
  OU
* NestJS (se necessário maior separação enterprise)

Base de dados:

* PostgreSQL

ORM:

* Prisma ORM

Autenticação:

* Auth.js / NextAuth

Storage:

* S3 ou Cloudinary

Realtime:

* Socket.io ou Supabase Realtime

Deploy:

* Vercel
* Railway / Supabase / Neon

A arquitetura deve ser:

* modular
* escalável
* segura
* fácil de manter
* preparada para mobile app futuro
